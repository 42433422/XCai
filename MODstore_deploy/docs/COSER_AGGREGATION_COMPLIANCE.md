# Coser AI 聚合平台运营要点

## 平台定位

Coser 不自训练大模型，作为统一聊天界面、模型路由、计费和风控中间层运行。厂商 API Key 只允许存放在后端服务器环境变量或密钥管理系统中，前端不得写入或展示官方密钥。

## 正式运营前置条件

- 使用企业主体注册火山引擎、百度智能云、阿里云百炼、腾讯云混元等厂商账号，并完成实名认证、企业认证和商用授权确认。
- 开通支付宝商户与微信支付商户，生产回调地址必须是公网 HTTPS。
- 接入短信服务商用于手机号验证码；本仓库默认 `COSER_SMS_PROVIDER=debug` 仅适合本地测试。
- 面向公众提供生成式 AI 服务时，按当地监管要求准备算法/大模型服务相关备案、用户协议、隐私政策、内容安全策略和投诉处理流程。

## 资金与风控

- 生产环境应设置 `PAYMENT_BACKEND=java`，Java 支付服务作为唯一钱包账本。
- LLM 调用必须先预授权余额，再调用厂商 API，成功后按 token 或最低价结算，失败时释放预授权。默认收费倍率为 `COSER_SERVICE_FEE_MULTIPLIER=1.5`，即每个模型的基础 token 成本自动乘以 1.5 后向用户扣费。
- 建议配置 Redis、单用户 RPM、单 IP RPM、单厂商 RPM、日/月全局消费上限，并在厂商控制台设置预算告警。
- 交易记录、模型调用日志、扣费流水、退款记录和风控事件至少保留 180 天，便于对账和争议处理。

## 密钥与部署

- `.env`、私钥、微信支付证书、支付宝私钥不得提交到 Git。
- 生产建议使用云厂商 Secret Manager 或容器编排密钥注入。
- BYOK 可作为会员/开发者能力开放；如只做平台统一密钥运营，可设置 `VITE_SHOW_BYOK=0` 隐藏入口。即使开放 BYOK，也应继续保留平台密钥和余额扣费链路，避免普通用户体验不稳定。
