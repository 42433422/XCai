# AI 数字管家 - 全局悬浮窗智能助手

## 一、产品定位

### 1.1 是什么

一个**全局悬浮窗形态的 AI 数字管家**，以极简小圆球形式存在于用户端，能听懂语音、看懂页面、代替用户操作，并通过平台的 **E-Skill 框架**实现**自主学习与自我进化**。

### 1.2 不是什么

- ❌ 不是简单的聊天机器人
- ❌ 不是被动执行指令的工具
- ❌ 不是用户购买的 AI Employee
- ❌ 不是固定功能的产品

### 1.3 对标产品

| 产品 | 可借鉴点 |
|------|----------|
| **腾讯云 AI 助手** | 智能、主动、理解上下文 |
| **OpenAI Operator** | 代替用户操作浏览器 |
| **Anthropic Computer Use** | 看懂屏幕、执行操作 |
| **Jarvis（钢铁侠）** | 主动建议、统筹全局 |

## 二、核心定位

### 2.1 角色定义

> **平台的"数字管家"——用户进来就有，不是买的，是自带的。**

| 维度 | 说明 |
|------|------|
| **与用户的关系** | 专属助手，了解用户习惯 |
| **与 AI Employee 的关系** | 平台的统一入口 + 超级员工 |
| **决策权** | AI 提议，用户拍板 |
| **执行权** | 操作类 AI 做，决策类（花钱）用户做 |

### 2.2 核心价值

```
省心：不用自己找按钮、填表单、研究流程
省力：动嘴就行，不用动手
省脑：AI 帮你想好该买什么、怎么用
```

## 三、核心能力

### 3.1 能力分层

| 层级 | 能力 | 说明 | 示例 |
|------|------|------|------|
| **L1 听懂** | 语音理解 + 视觉识别 | 听懂用户说的话，看懂当前页面 | "帮我买会员" → 理解意图 |
| **L2 想透** | 意图理解 + 上下文分析 | 不是机械执行，而是理解真实需求 | "我想要更多功能" → 推荐 Pro 会员 |
| **L3 做到** | 代替操作 | 点击、输入、导航、确认 | 自动导航到支付页，填好信息 |
| **L4 主动** | 预测需求 + 个性化建议 | 发现用户没说出口的需求 | "你经常搜索 XX，升级后体验更好" |
| **L5 进化** | 自主学习 + 自我升级 | 通过 E-Skill 框架给自己加能力 | 发现用户需要对比功能 → 自动编写对比技能 |

### 3.2 能力详解

#### L1：听懂

```
输入方式：
├── 语音（主要）：按住说话，转文字
├── 视觉（自动）：AI 自动截取当前屏幕，理解页面结构
└── 文本（备用）：打字输入

意图识别：
├── 快速识别（槽位提取）：从对话中提取关键信息
│   └── "帮我买 99 块的会员" → { intent: buy, item: member, price: 99 }
└── 深度理解（大模型）：分析真实意图
    └── "我想提升效率" → 分析用户现状 → 推荐解决方案
```

#### L2：想透

```
上下文理解：
├── 当前页面是什么？（首页、会员页、员工详情页...）
├── 用户当前状态？（会员等级、余额、使用频率...）
├── 用户历史行为？（买过什么、用过什么、搜索过什么...）
└── 用户真实需求是什么？（表面需求 vs 本质需求）

主动建议：
├── "你目前是免费版，升级到 Pro 可以解锁 XX 功能"
├── "你搜索的这个员工，还有一个更便宜的替代品"
└── "你的余额不足，需要先充值吗？"
```

#### L3：做到

```
操作能力：
├── 导航：跳转到指定页面
├── 点击：点击按钮、链接、选项
├── 输入：填写表单、搜索框
├── 选择：下拉菜单、单选/多选
├── 滚动：浏览长页面
└── 读取：获取页面内容、总结信息

操作原则：
├── 低风险操作：直接执行（如导航、读取）
├── 中风险操作：展示预览，用户确认后执行（如填写表单）
└── 高风险操作：必须用户确认（如支付、删除、修改密码）
```

#### L4：主动

```
行为分析：
├── 使用频率分析：哪些功能常用，哪些没用
├── 痛点发现：哪些地方用户经常卡住
├── 模式识别：用户的使用习惯和偏好
└── 需求预测：下一步用户可能需要什么

主动建议场景：
├── 额度预警："你本月免费搜索已用完 80%，升级后可无限使用"
├── 功能推荐："根据你的使用模式，XX 员工可能适合你"
├── 优化建议："你可以把 XX 员工添加到快捷方式，使用更方便"
└── 活动提醒："现在有 Pro 会员 8 折活动，要升级吗？"
```

#### L5：进化（预留）

```
进化路径：
1. AI 分析用户交互数据
2. 发现缺失能力（如用户经常问"对比员工"，但没有对比技能）
3. AI 通过 E-Skill 框架编写新技能
4. 管理端审核（可选）
5. 自动上线，AI 助手获得新能力

技术基础：
├── E-Skill 框架：技能 = 代码 + 配置 + 权限
├── Web Coding 能力：AI 可以写代码、改代码
├── 技能注册系统：新技能自动注册到 AI 助手
└── 数据反馈闭环：技能使用数据 → 分析 → 优化

⚠️ 当前状态：接口预留，暂不启动
```

| **等待确认** | 对话面板 + 确认弹窗 | 高风险操作，等待用户确认 |
| **完成态** | 对话面板 + 结果 | 操作完成，展示结果 |

## 六、E-Skill 框架集成

### 6.1 技能定义

```typescript
interface AgentSkill {
  id: string                    // 技能唯一标识
  name: string                  // 技能名称
  description: string           // 技能描述
  version: string               // 版本号
  
  // 触发条件
  trigger: {
    keywords?: string[]         // 关键词触发
    intent?: string[]           // 意图匹配
    context?: string[]          // 上下文匹配（页面路由）
  }
  
  // 执行逻辑
  execute: (context: AgentContext) => Promise<SkillResult>
  
  // 权限要求
  permission: 'read' | 'suggest' | 'execute' | 'full'
  
  // 元数据
  metadata: {
    author: string              // 作者（AI 或 管理员）
    created_at: number          // 创建时间
    evolution_count: number     // 进化次数
    usage_count: number         // 使用次数
  }
}
```

### 6.2 技能调用流程

```
1. 用户输入："帮我买 Pro 会员"
     ↓
2. 意图理解模块识别：{ intent: 'buy', target: 'pro_member' }
     ↓
3. E-Skill 运行时匹配技能：PaymentSkill
     ↓
4. 检查权限：需要 'execute' 权限（支付操作）
     ↓
5. 展示操作预览："即将购买 Pro 会员（¥99/月），确认吗？"
     ↓
6. 用户确认
     ↓
7. 执行技能：PaymentSkill.execute(context)
     ↓
8. 返回结果："购买成功"
```

### 7.2 权限分级

| 权限等级 | 能力 | 需要确认 | 示例 |
|----------|------|----------|------|
| **Read** | 读取页面内容 | 否 | 读取会员套餐信息 |
| **Suggest** | 建议操作 | 否 | 推荐升级会员 |
| **Execute** | 执行操作 | 用户确认后 | 填写表单、导航 |
| **Full** | 完全控制 | 必须确认 | 支付、删除、改密码 |

### 7.3 操作确认机制

```typescript
interface ActionPermission {
  action: string                // 操作类型
  risk: 'low' | 'medium' | 'high'
  
  // 确认策略
  confirmStrategy: 
    | 'auto'                    // 自动执行（低风险）
    | 'preview'                 // 展示预览，用户可取消（中风险）
    | 'explicit'                // 必须明确确认（高风险）
    
  // 确认提示
  confirmMessage: string        // "即将执行支付操作（¥99），确认吗？"
}
```

## 十、关键设计原则

### 10.1 用户决策权

```
AI 可以建议，但最终拍板的是用户。

❌ 错误做法：AI 直接付款，用户不知道
✅ 正确做法：AI 展示操作计划，用户确认后执行
```

### 10.2 透明度

```
AI 的每一步操作都要让用户知道。

❌ 错误做法：AI 默默操作，用户不知道发生了什么
✅ 正确做法：AI 展示操作状态，高亮操作元素
```

### 10.3 可撤销

```
用户随时可以停止 AI 操作。

❌ 错误做法：AI 操作到一半，用户无法中断
✅ 正确做法：操作面板有"停止"按钮
```

### 10.4 渐进式智能

```
从听懂指令 → 理解意图 → 主动建议 → 自我进化

不追求一步到位，而是逐步提升智能程度。
```

## 十一、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| **误操作** | AI 理解错误，执行了错误操作 | 操作前预览 + 用户确认 |
| **权限滥用** | AI 越权操作敏感功能 | 权限分级 + 敏感操作强确认 |
| **性能问题** | AI 操作导致页面卡顿 | 操作队列 + 异步执行 |
| **隐私泄露** | 用户数据被不当使用 | 隐私条款 + 本地存储 + 数据脱敏 |
| **用户不信任** | 用户不敢让 AI 操作 | 透明化操作日志 + 可撤销 |
