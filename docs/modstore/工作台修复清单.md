**当前状态**: 只有 system_prompt 文本框

**修复目标**:

├─ 行为规则

## 五、P3 - 长期优化

### 5.1 新旧版本整合

**当前问题**: V2 配置、旧版上传表单、Mod JSON 编辑器三套系统并行

**修复方案**:

```
方案 A：统一入口（推荐）
├─ 用户打开页面后，默认进入 V2 工作台
├─ 旧版上传表单折叠在"高级选项"中
├─ Mod JSON 编辑器折叠在"兼容模式"中
└─ 明确提示："推荐使用 V2 工作台，旧版功能仅用于兼容"

方案 B：分步引导
├─ 第一步询问："你要创建什么？"
│   ├─ 全新 AI 员工 → 进入 V2 工作台
│   ├─ 上传现有员工包 → 进入上传流程（解析到 V2）
│   └─ 编辑 Mod 声明 → 进入 Mod JSON 编辑器
└─ 后续所有数据统一存储为 V2 格式
```

**涉及文件**: `EmployeeAuthoringView.vue`

**实现建议**: 采用方案 A，工作量小，用户体验清晰

## 八、注意事项

1. **数据一致性**: 所有配置修改必须同步到 `employeeConfigV2`
2. **向后兼容**: 旧版员工包不能因为新增功能而失效
3. **性能优化**: 配置面板使用懒加载，避免一次性渲染所有模块
4. **类型安全**: 使用 TypeScript 类型定义，避免运行时错误
5. **用户体验**: 复杂配置使用折叠面板，避免页面过长
6. **测试覆盖**: 每个新组件编写单元测试
