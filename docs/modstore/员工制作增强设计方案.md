# 员工制作增强设计方案

## 一、现状分析

### 当前流程
```
上传员工包 → 填写名称/描述 → 沙盒测试 → 五维审核 → 填写行业/价格 → 上架
```

### 当前局限
- 员工能力定义完全依赖 `manifest.json` 和 `workflow_employees` 声明
- 缺少可视化的能力配置界面
- 无法直接配置声音克隆、技能模块、Agent 提示词等高级能力

---

## 二、增强目标

让员工制作页面成为一个**全能型 AI 员工配置中心**，支持以下能力模块的自由组合：

```
┌─────────────────────────────────────────┐
│           AI 员工能力配置中心             │
├─────────┬─────────┬─────────┬───────────┤
│ 基础信息 │ 工作流  │ 技能模块 │ 声音克隆  │
│ 身份定义 │ 编排引擎 │ Skill库  │ TTS/ASR  │
├─────────┼─────────┼─────────┼───────────┤
│ Agent   │ 知识库  │ API集成 │ 发布上架  │
│ 提示词  │ RAG配置 │ 第三方  │ 审核定价  │
└─────────┴─────────┴─────────┴───────────┘
```

## 三、新增能力模块设计

### 3.1 声音克隆模块（Voice Cloning）

#### 功能描述
允许为员工配置克隆声音，用于电话客服、语音播报等场景。

#### 数据结构
```json
{
  "voice_cloning": {
    "enabled": true,
    "voice_id": "vc_20240315_xxx",
    "voice_name": "客服小王",
    "source_type": "upload",
    "source_files": ["sample1.mp3", "sample2.mp3"],
    "settings": {
      "speed": 1.0,
      "pitch": 0,
      "emotion": "neutral",
      "language": "zh-CN"
    },
    "status": "trained"
  }
}
```

#### 页面交互
```
[步骤 X：声音配置]
├─ 启用声音克隆？ [开关]
│
├─ 声音来源
│   ├─ ○ 上传音频样本（需 3-10 分钟音频）
│   ├─ ○ 选择预设声音模板
│   └─ ○ 使用 API 训练新声音
│
├─ 声音参数调节
│   ├─ 语速：[滑块] 0.5x ~ 2.0x
│   ├─ 音调：[滑块] -10 ~ +10
│   ├─ 情感：[下拉] 中性/热情/专业/温和
│   └─ 语言：[下拉] 中文/英文/方言...
│
└─ 试听按钮 [▶ 播放样例]
```

#### API 接口
```
POST /api/voice-clone/train
  - 上传音频样本
  - 返回 voice_id 和训练状态

GET /api/voice-clone/{voice_id}/status
  - 查询训练进度

POST /api/voice-clone/{voice_id}/synthesize
  - 合成语音并返回音频流
```

### 3.2 Skill 技能模块（Skill Library）

#### 功能描述
为员工配置可复用的技能模块，如文档解析、数据分析、图像识别等。

#### 数据结构
```json
{
  "skills": [
    {
      "skill_id": "doc_analyzer",
      "skill_name": "文档分析器",
      "version": "2.1.0",
      "type": "document_processing",
      "config": {
        "supported_formats": ["pdf", "docx", "txt"],
        "extraction_mode": "full",
        "ocr_enabled": true,
        "language": "zh-CN"
      },
      "dependencies": ["ocr_engine", "pdf_parser"]
    },
    {
      "skill_id": "data_transformer",
      "skill_name": "数据转换器",
      "version": "1.5.0",
      "type": "data_processing",
      "config": {
        "input_format": "csv",
        "output_format": "json",
        "transform_rules": []
      }
    }
  ]
}
```

#### 页面交互
```
[步骤 X：技能配置]
├─ 已选技能列表
│   ├─ ┌──────────────────────────┐
│   │   │ ✓ 文档分析器 v2.1.0      │
│   │   │   支持 PDF/DOCX 解析     │
│   │   │   [配置] [移除]          │
│   │   └──────────────────────────┘
│   └─ ┌──────────────────────────┐
│       │ ✓ 数据转换器 v1.5.0      │
│       │   CSV → JSON 转换        │
│       │   [配置] [移除]          │
│       └──────────────────────────┘
│
├─ 添加技能 [按钮]
│   └─ 弹出技能市场选择器
│       ├─ 搜索：[输入框]
│       ├─ 分类：[全部] [文档] [数据] [图像] [音频]
│       └─ 技能列表（可多选）
│
└─ 技能依赖检查
    └─ 自动检测冲突和依赖关系
```

#### 内置 Skill 分类
| 分类 | 技能示例 | 描述 |
|------|---------|------|
| 文档处理 | 文档分析器、PDF 解析、OCR 识别 | 文档内容提取与理解 |
| 数据处理 | 数据转换器、格式校验、统计分析 | 数据清洗与转换 |
| 图像识别 | 验证码识别、物体检测、OCR | 图像内容分析 |
| 语音处理 | ASR 转写、TTS 合成、声纹识别 | 语音交互能力 |
| 网络请求 | API 调用、网页抓取、数据同步 | 外部系统集成 |
| 业务逻辑 | 条件判断、循环处理、异常处理 | 流程控制能力 |

### 3.3 Agent 提示词配置（Prompt Engineering）

#### 功能描述
为员工配置系统提示词、角色设定、行为规则等 Agent 核心逻辑。

#### 数据结构
```json
{
  "agent_config": {
    "system_prompt": "你是一个专业的文档分析助手，擅长从复杂文档中提取关键信息...",
    "role_definition": {
      "name": "文档分析员",
      "persona": "专业、细致、高效",
      "tone": "正式",
      "expertise": ["文档解析", "信息提取", "结构化处理"]
    },
    "behavior_rules": [
      {
        "rule_id": "rule_001",
        "name": "隐私保护",
        "description": "不输出敏感个人信息",
        "priority": "high",
        "action": "filter"
      },
      {
        "rule_id": "rule_002",
        "name": "格式规范",
        "description": "输出必须为 JSON 格式",
        "priority": "medium",
        "action": "enforce"
      }
    ],
    "few_shot_examples": [
      {
        "input": "请提取这份合同的关键条款...",
        "output": "{\"contract_type\": \"采购合同\", \"amount\": 50000, ...}"
      }
    ],
    "model_settings": {
      "model": "deepseek-chat",
      "temperature": 0.3,
      "max_tokens": 4000,
      "top_p": 0.9
    }
  }
}
```

#### 页面交互
```
[步骤 X：Agent 配置]
├─ 角色定义
│   ├─ 角色名称：[输入框] 文档分析员
│   ├─ 人设描述：[文本框] 专业、细致、高效...
│   ├─ 语调风格：[下拉] 正式/友好/专业/活泼
│   └─ 专业领域：[标签输入] 文档解析, 信息提取
│
├─ 系统提示词
│   └─ [多行文本编辑器]
│       支持变量：{user_name} {task_type} {context}
│       支持模板：导入预设模板...
│       字数统计：125/4000
│       [AI 优化提示词] [清空]
│
├─ 行为规则
│   ├─ 添加规则 [按钮]
│   │   └─ 规则名称：[输入]
│   │      规则描述：[输入]
│   │      优先级：[高/中/低]
│   │      执行动作：[过滤/强制执行/警告]
│   └─ 规则列表（可拖拽排序）
│
├─ 示例对话（Few-Shot）
│   ├─ 添加示例 [按钮]
│   │   ├─ 用户输入：[文本框]
│   │   └─ 期望输出：[文本框]
│   └─ 示例列表（至少 2 个）
│
└─ 模型参数
    ├─ 模型选择：[下拉] deepseek-chat / gpt-4 / ...
    ├─ Temperature：[滑块] 0.0 ~ 1.0
    ├─ Max Tokens：[数字输入]
    └─ Top P：[滑块] 0.0 ~ 1.0
```

### 3.4 知识库配置（Knowledge Base / RAG）

#### 功能描述
为员工关联知识库，支持 RAG（检索增强生成）能力。

#### 数据结构
```json
{
  "knowledge_base": {
    "enabled": true,
    "sources": [
      {
        "source_id": "kb_001",
        "name": "产品手册",
        "type": "document",
        "files": ["manual_v2.pdf", "faq.docx"],
        "chunk_size": 500,
        "overlap": 50
      },
      {
        "source_id": "kb_002",
        "name": "API 文档",
        "type": "url",
        "urls": ["https://docs.example.com/api"],
        "crawl_depth": 2
      }
    ],
    "retrieval_settings": {
      "strategy": "hybrid",
      "top_k": 5,
      "similarity_threshold": 0.75,
      "rerank_enabled": true
    }
  }
}
```

#### 页面交互
```
[步骤 X：知识库配置]
├─ 启用知识库？ [开关]
│
├─ 知识来源
│   ├─ 添加来源 [按钮]
│   │   ├─ 类型：[文件上传 / URL / 数据库 / API]
│   │   ├─ 文件上传：[拖拽区域]
│   │   └─ URL 输入：[输入框，支持批量]
│   └─ 来源列表
│       ├─ 产品手册（3 个文件）[编辑] [删除]
│       └─ API 文档（5 个页面）[编辑] [删除]
│
├─ 分块设置
│   ├─ 分块大小：[数字输入] 500 tokens
│   └─ 重叠大小：[数字输入] 50 tokens
│
└─ 检索配置
    ├─ 检索策略：[下拉] 向量/关键词/混合
    ├─ 返回数量：[数字输入] top 5
    ├─ 相似度阈值：[滑块] 0.0 ~ 1.0
    └─ 启用重排序：[开关]
```

## 七、可选配置说明

并非所有模块都是必填的，根据员工类型自由组合：

| 员工类型 | 必填模块 | 可选模块 |
|---------|---------|---------|
| 简单工作流员工 | 基础信息、工作流 | - |
| 对话型 Agent | 基础信息、Agent 配置 | 知识库、声音 |
| 技能型员工 | 基础信息、技能模块 | Agent 配置 |
| 电话客服员工 | 基础信息、声音克隆、Agent | 知识库、工作流 |
| 全能型员工 | 全部模块 | - |

## 八、迁移策略

### Phase 1：向后兼容
- 现有员工包继续工作，新增模块均为可选
- manifest 解析逻辑兼容旧结构
- 旧版员工可在编辑时逐步迁移到新结构

### Phase 2：渐进式增强
- 先实现 Agent 配置和 Skill 模块（最高频需求）
- 再实现声音克隆和知识库
- 最后完善 API 集成和高级功能

### Phase 3：生态建设
- 建立 Skill 市场
- 提供预设员工模板
- 支持员工能力组合推荐

---

## 九、总结

通过将员工制作从单一的工作流关联扩展为**多维度能力配置中心**，用户可以：

1. **灵活组合能力**：按需选择工作流、技能、声音、知识库等模块
2. **精细化配置**：每个模块都有详细的参数设置和测试验证
3. **可视化操作**：降低配置门槛，无需手写 manifest
4. **快速复用**：Skill 市场和预设模板加速制作流程

这使 MODstore 从"工作流商店"升级为真正的"AI 员工生态平台"。
