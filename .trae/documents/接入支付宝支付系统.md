# 接入支付宝：将 MODstore 升级为可支付的生产级网站

## 目标
在现有 MODstore 基础上接入支付宝支付，使用户能够：
1. 登录/注册
2. 浏览 MOD 市场
3. 通过支付宝购买 MOD/AI 员工
4. 查看订单与支付记录

## 现状分析
- **后端**：`modstore_server/app.py`（主 FastAPI 服务），已有用户认证、钱包系统
- **市场前端**：`market/`（Vue 市场页面），已有登录/注册/钱包/购买/管理功能
- **web 前端**：`web/`（Vue 工作台），已有登录/注册/工作台
- **支付宝包**：`alipay_package/` 已有完整的支付宝 SDK 封装：
  - `model_payment_alipay.py` — 支付宝核心 SDK（precreate/page/wap/refund/query/close/verify_notify）
  - `model_payment_compat.py` — FastAPI 路由层（checkout/notify/退款/诊断）
  - `model_payment_order_store.py` — 订单存储（JSON 文件落盘）

## 实施步骤

### 步骤 1：将支付宝包集成到后端服务

#### 1.1 复制支付宝模块到 modstore_server
- 将 `alipay_package/model_payment_alipay.py` 复制为 `modstore_server/alipay_service.py`
- 将 `alipay_package/model_payment_order_store.py` 复制为 `modstore_server/payment_orders.py`
- 调整模块路径引用（从 `backend.services` 改为相对导入）

#### 1.2 创建支付路由 `modstore_server/payment_api.py`
- 基于 `model_payment_compat.py` 创建 FastAPI 路由
- 端点：
  - `GET /api/payment/plans` — 获取可用套餐列表
  - `POST /api/payment/checkout` — 下单（支持 page/wap/precreate）
  - `POST /api/payment/notify/alipay` — 支付宝异步通知回调
  - `GET /api/payment/diagnostics` — 支付配置诊断
  - `GET /api/payment/entitlements` — 已购权益列表
  - `GET /api/payment/query/{out_trade_no}` — 交易查询
- 路由挂载到 `app.py`：`app.include_router(payment_router)`

#### 1.3 修改 `modstore_server/market_api.py`
- 在 `buyItem`（购买 MOD）接口中调用支付宝 checkout
- 或保留现有的余额购买 + 新增支付宝充值入口

### 步骤 2：前端增加支付页面

#### 2.1 修改 market 前端路由
- 在 `market/src/router/index.js` 中新增路由：
  - `/payment/plans` — 套餐页面（展示可购买的 MOD/套餐）
  - `/payment/checkout` — 收银台（选择支付方式，显示支付宝二维码/跳转链接）
  - `/payment/order/{id}` — 订单状态页（轮询支付状态）

#### 2.2 创建 `market/src/views/PaymentPlansView.vue`
- 展示可购买的套餐列表（从 `/api/payment/plans` 获取）
- 每个套餐显示价格、描述、"购买"按钮
- 点击购买后调用 `/api/payment/checkout`
- 根据返回类型处理：
  - `page` 类型：跳转 `redirect_url` 到支付宝
  - `wap` 类型：跳转 `redirect_url`
  - `precreate` 类型：生成二维码展示

#### 2.3 创建 `market/src/views/PaymentCheckoutView.vue`
- 处理 checkout 响应
- precreate 模式：展示二维码（使用 `qrcode` 库或 canvas 生成）
- 轮询 `/api/payment/query/{order_id}` 检查支付状态
- 支付成功后跳转到订单完成页

#### 2.4 修改 `market/src/api.js`
- 新增支付相关 API：
  ```js
  paymentPlans: () => req('/api/payment/plans'),
  paymentCheckout: (planId) => req('/api/payment/checkout', { method: 'POST', body: JSON.stringify({ plan_id: planId }) }),
  paymentQuery: (orderId) => req(`/api/payment/query/${orderId}`),
  paymentDiagnostics: () => req('/api/payment/diagnostics'),
  paymentEntitlements: () => req('/api/payment/entitlements'),
  ```

#### 2.5 修改 `market/src/App.vue`
- 导航栏添加"充值"或"套餐"入口
- 登录状态下显示用户余额

### 步骤 3：支付宝配置指引

#### 3.1 创建 `.env` 配置文件
- 在 `MODstore_deploy/` 根目录创建 `.env.example`：
  ```
  ALIPAY_APP_ID=你的应用ID
  ALIPAY_APP_PRIVATE_KEY=你的应用私钥（RSA2格式PEM）
  ALIPAY_ALIPAY_PUBLIC_KEY=支付宝公钥
  ALIPAY_NOTIFY_URL=https://你的域名/api/payment/notify/alipay
  ALIPAY_DEBUG=1  # 沙箱模式
  ```

#### 3.2 提供沙箱测试指引
- `ALIPAY_DEBUG=1` 时自动使用支付宝沙箱网关
- 可在支付宝开放平台申请沙箱账号测试

### 步骤 4：整合市场前端与支付

#### 4.1 修改 `market/src/views/HomeView.vue`
- 现有购买按钮改为调用支付宝 checkout
- 展示支付方式和价格
- 已购商品显示"已拥有"

#### 4.2 修改 `market/src/views/WalletView.vue`
- 新增支付宝充值入口（调用 checkout API）
- 显示充值记录和余额变动

### 步骤 5：后端增强

#### 5.1 用户与支付订单关联
- 在 `models.py` 新增 `PaymentOrder` 模型（关联 user_id）
- 或复用现有的 JSON 订单存储，增加 user_id 字段

#### 5.2 支付成功后自动发放权益
- 在 notify 回调中：验签 → 更新订单状态 → 发放 MOD 访问权限
- 复用 `user_owns_mod` 逻辑

#### 5.3 修改 `market_api.py` 的 `buyItem`
- 免费 MOD：直接发放
- 付费 MOD：返回支付宝 checkout 信息

## 技术决策

### 支付模式选择
- **precreate（当面付/订单码）**：适合展示二维码，用户扫码支付
- **page.pay（电脑网站支付）**：适合 PC 端跳转支付宝网页
- **wap.pay（手机网站支付）**：适合移动端
- **自动选择**：根据 User-Agent 自动切换，已有代码实现

### 订单存储
- 开发阶段：复用 `payment_orders.py` 的 JSON 文件存储
- 生产环境：建议迁移到 SQLite 或 PostgreSQL（已有 `models.py` 基础）

### 安全
- 异步通知必须验签（已实现 `verify_notify`）
- 金额校验（已实现 `amount_mismatch` 检查）
- 幂等处理（已实现 `already_paid` 处理重复通知）

## 依赖
- `python-alipay-sdk`：需要 `pip install python-alipay-sdk`
- `qrcode`（可选）：前端展示 precreate 二维码
- 前端无需额外依赖，支付宝跳转和轮询均可原生实现

## 风险与注意事项
1. 支付宝公钥 ≠ 应用公钥，配置时注意区分
2. `ALIPAY_NOTIFY_URL` 必须是公网可访问的 HTTPS URL（本地开发可用 ngrok 穿透）
3. 沙箱模式下使用 `ALIPAY_DEBUG=1`，生产环境必须移除
4. 订单超时：支付宝默认 15 分钟未付款自动关闭
