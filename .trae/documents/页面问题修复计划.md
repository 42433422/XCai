# 页面问题修复计划

## 问题分析
1. **"加载员工失败: Field required"** - EmployeeAuthoringView.vue 调用 `api.catalog` 时 API 返回错误
2. **"div 少了登录状态"** - 用户未登录或登录状态丢失，导航栏显示登录/注册而非用户信息
3. **"管理员和应用页面不见了"** - 可能是管理员路由或组件问题

## 修复步骤

### 1. 修复 EmployeeAuthoringView.vue 的 API 调用
- 文件：`src/views/EmployeeAuthoringView.vue`
- 问题：`api.catalog('', 'employee_pack', 100, 0)` 调用时 API 可能需要认证或参数不完整
- 修复：检查 API 参数，确保认证令牌正确传递

### 2. 恢复登录状态显示
- 确认 App.vue 中的 `isLoggedIn` 状态正确工作
- 如果用户已登录但显示未登录，检查 `api.me()` 调用和 token 处理

### 3. 检查管理员和应用页面路由
- 确认 `/admin/database` 路由存在
- 检查是否有 Application/应用页面需要添加

### 4. 验证后端 API 状态
- 确认后端服务器正在运行（当前在 8001 端口）
- 检查 `/api/market/catalog` 接口是否正常

## 当前路由状态
```
/my-store         → MyStoreView.vue (购买历史)
/workbench        → EmployeeAuthoringView.vue (员工制作，默认)
/workbench/repository → RepositoryView.vue (仓库)
/workbench/workflow   → WorkflowView.vue (工作流)
/ai-store         → AiStoreView.vue
/admin/database   → AdminDatabaseView.vue
```
